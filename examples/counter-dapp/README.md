# Counter dApp - ADI Chain Example

A minimal on-chain counter dApp deployed to ADI Chain using **Hardhat 3** and a single HTML frontend.

This is the default project you get when you run `npx create-adi-app` and choose the Hardhat template.

## What it does

- Stores a single `uint256` on-chain
- `increment()` - adds 1 (anyone can call)
- `setNumber(n)` - sets an arbitrary value (anyone can call)
- Frontend reads the current value on load; lets you connect MetaMask to write

## Project structure

```
counter-dapp/
├── contracts/
│   └── Counter.sol          ← Solidity contract
├── frontend/
│   └── index.html           ← Single-file dApp UI (ethers.js via CDN)
├── ignition/modules/
│   └── Counter.ts           ← Hardhat Ignition deployment module (optional)
├── deploy.mjs               ← Recommended deploy script (ethers.js direct)
├── hardhat.config.ts        ← Hardhat 3 config with ADI networks pre-injected
├── .env.example             ← Environment variable template
└── package.json
```

## Quick start

### 1. Install dependencies

```bash
npm install
```

### 2. Configure your wallet

```bash
cp .env.example .env
# Edit .env - set TESTNET_PRIVATE_KEY to your wallet's private key
```

Get free testnet ADI from the faucet before deploying:
```
http://faucet.ab.testnet.adifoundation.ai
```

### 3. Compile

```bash
npm run compile
```

### 4. Deploy

```bash
npm run deploy
# Output: ✔ Counter deployed to: 0x...
```

### 5. Update the frontend

Open `frontend/index.html` and paste your deployed address at **line 56**:

```javascript
const CONTRACT_ADDRESS = "0xYourDeployedAddressHere";
```

### 6. Serve the frontend

```bash
npx serve frontend
# Open http://localhost:3000 (must be HTTP - MetaMask blocks file://)
```

## Deploying to mainnet

```bash
# Make sure MAINNET_PRIVATE_KEY is set in .env
npm run deploy:mainnet
```

## Why `deploy.mjs` instead of `hardhat ignition deploy`?

The ADI testnet RPC does not support the `pending` block tag that Hardhat Ignition uses for nonce synchronisation. The `deploy.mjs` script deploys via ethers.js directly - no such limitation, no extra config needed.
